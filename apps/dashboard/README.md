## Email Signature Testing

For reply-signature changes, test composition locally before using a real mailbox:

```bash
npm run test:email
```

This covers:

- signature deduplication in plain text replies
- HTML escaping and branded signature injection
- Gmail multipart MIME assembly and base64url encoding
- CID-based inline signature image embedding for supported signatures

Recommended workflow:

1. Run `npm run test:email` locally while iterating on signatures.
2. Use one stable test mailbox for occasional end-to-end verification.
3. Keep Outlook live tests low-volume and avoid automated account-to-account loops.

If your signature HTML contains `{{logo_url}}`, the send flow will replace it with an inline embedded image when a logo has been uploaded for that account.

You can start editing the page by modifying `app/page.tsx`. The page auto-updates as you edit the file.
